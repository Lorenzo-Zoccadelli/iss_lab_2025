Introduction

Si vuole realizzare una GUI per il gioco "Conway's Game of Life" già realizzato in java.

Requirements

ConwayLife25 una GUI per Java

Progetto conwaygui: realizzazione di una pagina HTML che funga da dispostivo di input-ouput evoluto per l’applicazione ConwayLife25 in Java.

La pagina HTML svolge il ruolo di:

  • un dispositivo di input, in quanto permette a un utente umano di settare lo stato iniziale delle celle e di inviare comandi di gestione dell’applicazione con i pulsanti start/stop/exit.

  • un dispositivo di output, in quanto permette di visualizzare lo stato della griglia

I files relativi alla sua realizazione sono:

  • guipage.html: pagina HTML

  • WSConwayguiLifeMqtt: gestione delle WebSocket lato server

  • iomap.js: mappa con le celle

  • commsocket.js: gestione della WebSocket lato client

Requirement analysis

I requisiti richiedono la realizzazione dell'interfaccia grafica con le seguenti caratteristiche.
Funzionalità di input:
  • Possibilità di impostare la configurazione iniziale della griglia
  • Un pulsante "Start" che permetta di far partire la simulazione
  • Un pulsante "Stop" che permetta di mettere in pausa la simulazione
  • Un pulsante "Exit" che permetta di uscire dall'applicazione e terminare la simulazione

Funzionalità di output:
  • La griglia che rappresenta lo stato del gioco dovrà essere rappresentata come nella figura riportata per mostrare il progredire del gioco

Vincoli:
  • L'interfaccia grafica dovrà essere realizzata attraverso una pagina HTML, perciò tutti i pulsanti di controllo e la griglia dovranno essere realizzati con l'utilizzo di HTML, CSS e JS in modo da poter fornire un'interfaccia grafica web

Punti aperti:
  • Una volta dopo aver premuto il pulsante stop la simulazione potrà riprendere? E se si, Utilizzando nuovamente il pulsante start?
  • Il pulsante exit dovrà uscire anche dalla pagina o semplicemente fermerà in maniera definitiva la simulazione? In ogni caso, la simulazione dovrà essere resettata?

Problem analysis

Dovrà essere realizzata un'interfaccia grafica in html per l'applicazione in java. Perciò verranno potranno essere utilizzate delle websocket per la comunicazione tra client (pagina html) e server (applicazione java). Sarà quindi necessaria una comunicazione tra i due attraverso opportuni moduli (client websocket in js e server websocket in java) che comunicheranno attraverso la rete. Le interazioni di input previste (attraverso la comunicazione appena descritta) saranno:
  • Pressione di una cella della griglia: cambiamento dello stato della cella corrispondente della griglia
  • Pressione del pulsante "start": modifica dello stato del controller java e inizio della simulazione
  • Pressione del pulsante "stop": modifica dello stato del controller java e interruzione della simulazione
  • Pressione del pulsante "exit": modifica dello stato del controller java e terminazione della simulazione con reset

Per quanto riguarda l'output, una volta avviata la simulazione, questa avrà un comportamento ciclico, con un aggiornamento periodico automatico dello stato della griglia. In caso di pausa della simulazione l'aggiornamento verrà sospeso, salvo riprendere in caso di nuova pressione del pulsante "start". Perciò, a ogni agiornamento della griglia, dovrà essere comunicato, sempre attraverso websocket, dal server java la nuova configurazione della griglia alla gui client.
